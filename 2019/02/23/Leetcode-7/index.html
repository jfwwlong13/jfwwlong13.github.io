[LeetCode] 7. Reverse Integer

发表于 2019-02-23 | 分类于 LeetCode | 评论数：

Problem

Given a 32-bit signed integer, reverse digits of an integer.
Example 1:

Input: 123
Output: 321

Example 2:

Input: -123
Output: -321

Example 3:

Input: 120
Output: 21

Note: Assume we are dealing with an environment which could only store integers within the 32-bit signed integer range: [−231, 231 − 1]. For the purpose of this problem, assume that your function returns 0 when the reversed integer overflows.

Thinking

思路很简单。就是反着来。每次取出x的余数加到当前的结果里。这道题的主要难点在于溢出的判断。
当然比较偷懒的思路是用long来存数字，这样就不会溢出。但是面试的时候肯定不会让你这么用，因为这个就是他们考察的点。

Solution

对于这种判断溢出的题目，一个比较经典的判断方法就是用当前的结果和下一个数字来判断是否溢出。具体方法参考代码。

时间复杂度：O(\(log_{10}X\))。X是数字的大小
空间复杂度：O(1)
