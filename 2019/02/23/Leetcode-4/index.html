[LeetCode] 4. Median of Two Sorted Arrays

发表于 2019-02-23 | 分类于 LeetCode | 评论数：

Problem

There are two sorted arrays nums1 and nums2 of size m and n respectively.
Find the median of the two sorted arrays. The overall run time complexity should be O(log (m+n)).
You may assume nums1 and nums2 cannot be both empty.

Example 1: nums1 = [1, 3]
nums2 = [2]
The median is 2.0

Example 2:
nums1 = [1, 2]
nums2 = [3, 4]
The median is (2 + 3)/2 = 2.5

Thinking

这道题还是很有难度的。
因为是两个有序数组，所以第一时间想到的就是二分搜索。
由于两个数组的和L可能是奇数也可能是偶数，所以我们需要分两种情况讨论：

  1. 当L是奇数时：返回第\(\frac{L}{2} + 1\)大的数
  2. 当L是偶数时，返回第\(\frac{L}{2}\)和\(\frac{L}{2} + 1\)的平均数

这样我们就可以有一个子函数，用来找两个数组中第K大的数。
那具体怎么找呢？我们可以分别取两个数组中第K / 2大的数M1和M2，通过比较M1和M2来删除部分元素：

  1. 如果M1 < M2，那么我们可以丢弃nums1数组中的包括M1以及其之前的所有元素
  2. 如果M1 > M2，则抛弃M2中同样长度的元素
  3. 如果M1 = M2，则直接返回M1或者M2

为什么可以这样删除元素呢？因为如果M1 < M2，那么对于nums1数组，M1及其之前的元素永远不可能成为两个数组中第K大的数。这个并不难想。所以我们就可以这样一直递归下去。

Solution

根据上面的思路就可以写代码了。
这里有一点要注意的是，由于数组nums1的长度有可能小于K / 2，比如[0]和[1,2,3,4,5]，这个时候对于nums1数组，我们需要取K / 2和nums1的长度的最小值，防止数组越界。

时间复杂度：O(\(log(M + N)\))。M和N分别是nums1和nums2的长度
空间复杂度：O(\(log(M + N)\))。空间主要用在了递归上面。如果不算递归的空间那么空间复杂度就是O(1)
